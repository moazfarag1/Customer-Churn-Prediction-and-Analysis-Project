# GitHub Content Report

This document outlines the contents of the public GitHub version for the Customer Churn Prediction project. This repository has been cleaned and optimized for public distribution, ensuring that it looks and functions as a professional student project.

## Main Components Included

- **`README.md`**: Comprehensive overview of the project, including setup and run instructions.
- **`requirements.txt`**: Standardized dependencies for the project environment.
- **`app/`**: Contains the source code for the interactive Streamlit dashboard (`app.py`), allowing users to run the final model locally.
- **`assets/`**: High-quality, polished plots (EDA, feature importance, confusion matrices, ROC curves) to be showcased in documentation.
- **`data/`**: Anonymized, non-sensitive raw dataset (`telco_customer_churn.csv`), cleaned data artifacts, and statistical summaries required to reproduce the analysis.
- **`models/`**: Serialized model pipelines and feature schema (`.pkl` files) required to run inferences without retraining.
- **`notebooks/`**: The core execution pipeline, cleanly structured:
  - `01_moaz_eda_preprocessing.ipynb`
  - `02_mohy_feature_engineering.ipynb`
  - `03_mahmoud_modeling_baseline.ipynb`
  - `04_ali_xgboost_optimization.ipynb`
- **`MODEL_CARD.md`**: Standardized model card detailing the intended use, metrics, and data.
- **`RUN_PROJECT_GUIDE.md`**: Step-by-step guide for developers or reviewers to reproduce the environment and results.

All internal team guides, viva preparation, and AI-generated analysis files have been successfully excluded from this public distribution.
